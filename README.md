# Анализ эмоционального окраса русскоязычных текстов

Проект выполнен в рамках выпускной квалификационной работы на тему  
**«Анализ эмоционального окраса текстов методами машинного обучения»**.

Работа посвящена задаче автоматической классификации коротких русскоязычных текстов из социальных сетей по эмоциональному окрасу. Рассматривается трехклассовая постановка задачи:

- `negative` — отрицательный эмоциональный окрас;
- `neutral` — нейтральный эмоциональный окрас;
- `positive` — положительный эмоциональный окрас.

В рамках проекта сравниваются классические модели машинного обучения, трансформерная модель и ансамблевые подходы.

## Цель проекта

Цель проекта — исследовать методы автоматического анализа эмоциональной окраски русскоязычных текстов и сравнить качество различных классификационных моделей в рамках единой экспериментальной схемы.

## Использованные данные

В работе использовались два русскоязычных корпуса сообщений из социальных сетей:

| Датасет | Источник текстов | Роль в работе | Используемые классы | Train | Val | Test | Всего |
|---|---|---|---|---:|---:|---:|---:|
| RuSentiTweet | Twitter | основной корпус для экспериментов E0–E6, E8, E9 | `negative`, `neutral`, `positive` | 7 957 | 885 | 2 211 | 11 053 |
| RuSentiment | VKontakte | дополнительный корпус для E7 и E8 | `negative`, `neutral`, `positive` | 18 920 | 2 103 | 2 214 | 23 237 |

В исходных корпусах также присутствовали классы `speech` и `skip`, однако они были исключены, так как не отражают непосредственно положительную, отрицательную или нейтральную тональность текста.

## Предобработка данных

В проекте не применялась агрессивная очистка текстов: ссылки, упоминания, эмодзи, пунктуация и сленг не удалялись отдельной функцией, поскольку такие элементы могут быть информативными для анализа эмоционального окраса сообщений из социальных сетей.

Основные этапы подготовки данных:

- приведение названий столбцов к единому формату `text` и `label`;
- фильтрация целевых классов;
- проверка пропущенных значений;
- удаление дубликатов;
- стратифицированное разделение данных на train, validation и test;
- кодирование меток для трансформерной модели.

## Признаковые представления

Для классических моделей использовались TF-IDF-признаки:

| Представление | Описание |
|---|---|
| Word TF-IDF | словные n-граммы с диапазоном `(1, 2)` |
| Char TF-IDF | символьные n-граммы с диапазоном `(3, 5)` |
| Combined TF-IDF | объединение word TF-IDF и char TF-IDF |

Для трансформерной модели использовалась токенизация с помощью `AutoTokenizer` для модели `cointegrated/rubert-tiny2`. Максимальная длина последовательности составляла 128 токенов.

## Исследованные модели

В проекте рассматривались следующие группы моделей.

### Базовая модель

- `DummyClassifier`

### Классические модели машинного обучения

- `MultinomialNB`
- `BernoulliNB`
- `ComplementNB`
- `LogisticRegression`
- `LinearSVC`
- `SVC(kernel='rbf')`
- `RidgeClassifier`
- `SGDClassifier`

### Трансформерная модель

- `cointegrated/rubert-tiny2`

### Ансамблевые модели

- ансамбли классических моделей:
  - hard voting;
  - soft voting;
  - weighted soft voting;
- гибридный ансамбль:
  - классическая модель + ruBERT-tiny2.

## План экспериментов

| Эксперимент | Содержание | Цель |
|---|---|---|
| E0 | Baseline: TF-IDF + классические модели + ruBERT-tiny2 | Получить базовые результаты |
| E1 | Расширение классических моделей | Сравнить разные семейства алгоритмов |
| E2 | Word TF-IDF и char TF-IDF | Проверить влияние признакового пространства |
| E3 | Combined word+char TF-IDF | Оценить эффект объединения признаков |
| E4 | `class_weight='balanced'` | Проверить влияние балансировки классов |
| E5 | Ансамбли классических моделей | Исследовать устойчивость ансамблей |
| E6 | Hybrid classical+BERT | Объединить классическую и трансформерную модели |
| E7 | RuSentiment in-domain | Проверить модели на втором корпусе |
| E8 | Cross-dataset transfer | Исследовать перенос между датасетами |
| E9 | Error analysis | Выявить типичные ошибки классификации |

## Метрики качества

Основной метрикой сравнения моделей является `weighted F1`, так как в наборах данных присутствует дисбаланс классов.

Дополнительно фиксировались:

- `accuracy`;
- `weighted precision`;
- `weighted recall`;
- `macro F1`;
- `train_time_sec`.

## Основные результаты

| Эксперимент | Ключевой результат | Интерпретация |
|---|---|---|
| E0 | ruBERT-tiny2 tuned: weighted F1 = 0.6958 | Трансформерная модель превзошла базовые классические подходы |
| E2 | SGDClassifier hinge + char TF-IDF: weighted F1 = 0.6741 | Символьные признаки оказались лучшими для классических моделей |
| E3 | Лучший combined TF-IDF: weighted F1 = 0.6622 | Объединение word+char не превзошло char-only |
| E5 | Лучший классический ансамбль: около weighted F1 = 0.6686 | Ансамбли стабильны, но не превосходят лучшую одиночную модель |
| E6 | Hybrid classical+BERT: weighted F1 = 0.7232, macro F1 = 0.707 | Лучший результат всей работы |
| E7 | RuSentiment: BERT = 0.7698, SGDClassifier = 0.7660 | Качество сопоставимо, классическая модель быстрее |
| E8 | Снижение weighted F1 при переносе между корпусами | Кросс-датасетный перенос является сложной задачей |
| E9 | Основная ошибка: `negative → neutral` | Негативная окраска часто выражается неявно |

Лучший результат всей работы был получен гибридным ансамблем, объединяющим классическую модель на char TF-IDF-признаках и трансформерную модель ruBERT-tiny2.

## Анализ ошибок

В рамках анализа ошибок были выделены основные причины неверной классификации:

- неявно выраженная эмоциональная окраска;
- сарказм и ирония;
- сленг и нестандартное написание;
- неоднозначный контекст;
- краткость сообщения;
- смешанная эмоциональная окраска.

Наиболее частой ошибкой стала классификация отрицательных сообщений как нейтральных: `negative → neutral`.

## Установка и запуск

1. Клонировать репозиторий:

```bash
git clone https://github.com/dishubnyakova/sentiment-analysis-ru.git
cd sentiment-analysis-ru
```

2. Создать и активировать виртуальное окружение:

```bash
python -m venv venv
```

Для Linux/macOS:

```bash
source venv/bin/activate
```

Для Windows:

```bash
venv\Scripts\activate
```

3. Установить зависимости:

```bash
pip install -r requirements.txt
```

4. Открыть Jupyter Notebook:

```bash
jupyter notebook
```

5. Запустить нужные ноутбуки из папки `notebooks/`.

## Вывод

Проведенное исследование показало, что качество анализа эмоционального окраса русскоязычных коротких текстов зависит от типа признакового представления, модели и домена данных.

Среди классических моделей наиболее эффективным оказалось сочетание `SGDClassifier` с функцией потерь `hinge` и символьного TF-IDF-представления. Трансформерная модель ruBERT-tiny2 показала более высокое качество по сравнению с отдельными классическими моделями, однако лучшим подходом стал гибридный ансамбль, объединяющий классическую модель и BERT.

Кросс-датасетный перенос между RuSentiTweet и RuSentiment показал, что модели чувствительны к источнику данных и стилю текстов. Анализ ошибок подтвердил, что наиболее сложными остаются случаи сарказма, иронии, сленга, неявной тональности и смешанной эмоциональной окраски.
